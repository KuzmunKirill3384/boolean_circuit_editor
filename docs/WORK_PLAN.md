## Кузьмин Кирилл Владимирович: Бэкенд (модель, команды, логика)

### Задачи (порядок удобный для параллельной работы с фронтом)

1. **Модель схемы (Composite)**
   - Интерфейс/абстрактный класс узла (вход, выход, логический элемент), метод `accept(Visitor)`.
   - Реализация элементов: AND, OR, XOR, EQUAL.
   - Схема как граф узлов и связей (входы/выходы).

2. **Валидация**
   - Проверка на циклы при добавлении/редактировании связей.
   - Проверка количества входов/выходов.
   - Понятные сообщения об ошибках для UI.

3. **Command, Factory, Visitor**
   - Интерфейс команды: `execute()`, `undo()` (и при необходимости `redo()`).
   - Команды: добавление/удаление элемента, создание/удаление связи, перемещение.
   - Factory создания команд по типу операции.
   - Visitor для обхода модели в командах (do/undo).
   - История команд (Undo/Redo).

4. **Формат файлов (Builder)**
   - Реализация Builder’а: сборка модели из XML, выгрузка модели в XML.
   - Следование спецификации формата из `docs/FILE_FORMAT.md` (ведёт Боос, согласование с Кузьминым).

5. **Вычислительная логика**
   - Генерация таблицы истинности по схеме (и по выбранному элементу).
   - Список узлов, влияющих на выбранный элемент (для подсветки на фронте).
   - Упрощение при фиксированных входах: оценка числа удаляемых элементов, само упрощение.
   - Полиномиальное представление (например, Жегалкин) для каждого элемента.

### Паттерны

- **Composite** — иерархия элементов схемы.
- **Command** — операции редактирования с отменой.
- **Factory** — создание команд.
- **Visitor** — обход модели в командах.
- **Builder** — загрузка/сохранение XML.

### Критерии готовности

- Загрузка/сохранение схемы в XML.
- Все изменения модели через команды; Undo/Redo работают.
- API для таблицы истинности, упрощения и полиномов готовы к вызову из UI.

## Климаш Иван Витальевич: Фронтенд (UI)

### Задачи (параллельно с бэком после контрактов)

1. **Базовый UI**
   - Рабочая область: отрисовка элементов и связей.
   - Добавление элементов (AND, OR, XOR, EQUAL), рисование связей (вызов команд бэка).
   - Перемещение элементов (через команды).
   - Вызов валидации перед применением операций, отображение ошибок.

2. **Таблица истинности**
   - Отображение таблицы для всей схемы (данные с бэка).
   - Режим по элементу: выбор узла → запрос к бэку → отображение таблицы и подсветка влияющих элементов на схеме.

3. **Упрощение схемы**
   - Ввод фиксированных значений на входах.
   - Отображение оценки «сколько элементов можно убрать» (данные с бэка).
   - Кнопка упрощения и обновление схемы после ответа бэка.

4. **Полиномиальное представление**
   - Отображение полинома для выбранного элемента (данные с бэка): панель, подсказка или отдельное окно.

5. **Настройки отображения**
   - Цвета, размеры элементов, шрифты; сохранение/загрузка настроек.

6. **Сохранение и загрузка**
   - Кнопки/меню «Сохранить в XML» и «Загрузить из XML», диалоги выбора файла, вызов API бэка, обработка ошибок.

### Критерии готовности

- Все сценарии ТЗ доступны из UI и работают с бэком.
- Удобное руководство оператора можно составить по текущему интерфейсу (скриншоты делает Боос).
