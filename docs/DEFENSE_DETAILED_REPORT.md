# Детальный файл для защиты проекта

## 1) Паспорт проекта

- **Проект:** Редактор булевых схем
- **Репозиторий:** `boolean_circuit_editor`
- **Цель:** создание, редактирование и анализ булевых схем с экспортом/импортом XML
- **Целевая архитектура по ТЗ:** `Composite + Command + Visitor + Factory + Builder`
- **Команда (из README):**
  - Боос Дмитрий Алексеевич
  - Климаш Иван Витальевич
  - Кузьмин Кирилл Владимирович

## 2) Что реализовано на текущий момент (фактическое состояние кода)

### 2.1 Основной функционал редактора

- Добавление узлов на поле: `IN`, `OUT`, `AND`, `OR`, `XOR`, `EQUAL`
- Соединение узлов по пинам (проверка валидности соединения)
- Разрыв соединений
- Удаление блоков (через клавиши `Delete`/`Backspace`, меню, тулбар)
- Перемещение по доске:
  - колесо мыши: zoom
  - стрелки клавиатуры: панорамирование без мыши
  - средняя кнопка мыши: hand-pan
- Таблица истинности для всей схемы
- Подсветка при построении связи (выбранный исходный пин)
- Настройки визуализации (диалог настроек)
- Сохранение и загрузка схемы в XML

### 2.2 Backend (актуально после рефакторинга)

- `src/backend/model/circuit.py`
  - модель схемы (узлы/связи)
  - проверки структуры
  - валидация соединений
  - контроль циклов
- `src/backend/commands/history.py`
  - undo/redo стек
  - команды добавления/удаления/перемещения/соединения/разъединения
- `src/backend/application/app_controller.py`
  - фасадный слой между GUI и логикой
  - загрузка/сохранение
  - вызовы truth-table логики
  - безопасная обработка ошибок расчета
- `src/backend/io/xml_builder.py`
  - импорт/экспорт XML
- `src/backend/logic/truth_table.py`
  - расчет таблиц истинности
  - расчет по узлу (в коде доступно, в UI сейчас отключено)
  - оценка/упрощение/полиномы (логика есть, в UI часть функций скрыта)

### 2.3 Frontend

- `src/frontend/app/main_window.py`
  - меню, тулбар, доки, подсказки, действия пользователя
- `src/frontend/canvas/scene.py`
  - графическая сцена: узлы, пины, линии, режимы работы
- `src/frontend/canvas/view.py`
  - управление камерой/масштабом и клавиатурной навигацией

## 4) Сопоставление с требованиями ТЗ

### 4.1 Функции

1. **Создание и редактирование схемы** — реализовано в рабочем виде.
- Добавление/удаление узлов, соединение/разъединение, перемещение по сцене.
- Undo/Redo есть.
- Проверка корректности структуры (циклы, пины, связи) есть.

2. **Таблица истинности** — частично/в основном реализовано.
- Для всей схемы: да.
- Для отдельного элемента с подсветкой влияющих: логика есть, но в текущем UI отключено.

3. **Упрощение схемы** — логика есть, UI сейчас упрощен.
- В backend функции присутствуют.
- В текущем пользовательском интерфейсе отключено (по последним решениям по UX).

4. **Полиномиальное представление** — логика в backend есть, UI отключен.

5. **Настройка изображения** — реализовано (цвета, размеры, шрифт через settings диалог).

6. **Сохранение/загрузка XML** — реализовано.

### 4.2 Паттерны (по факту)

- **Command** — реализован явно (команды + история undo/redo).
- **Builder** — реализован явно (`xml_builder`).
- **Composite** — реализован частично (модель схемы как граф объектов), но не оформлен как классическая иерархия Component/Composite/Leaf.
- **Visitor** — в текущем коде явно как паттерн не выделен (требует формализации).
- **Factory команд** — в явном виде отдельной фабрики пока нет (создание команд происходит в контроллере).
