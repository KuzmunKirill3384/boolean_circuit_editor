# Рабочие заметки по проекту (расширенная версия)

> Внутренний материал для подготовки к защите.  
> Цель: иметь максимально подробную фактуру по реализации, ролям, рискам и оставшимся задачам.

---

## 1. Контекст проекта

### 1.1 Тема

Проект: редактор булевских схем с возможностью:
- конструирования схемы из базовых логических элементов;
- проверки корректности соединений;
- вычисления таблицы истинности;
- сохранения/загрузки схемы в XML;
- (по полной версии ТЗ) упрощения и полиномиального представления.

### 1.2 Команда и роли (базовая матрица)

- **Климаш Иван** — фронтенд/GUI, сценарии взаимодействия.
- **Кузьмин Кирилл** — backend, модель, вычислительная логика, архитектурная стабилизация.
- **Боос Дмитрий** — документация, отчётные артефакты, тестовая и методическая часть.

### 1.3 Что важно для защиты

На защите обычно проверяют:
1) работоспособность пользовательского сценария;
2) соответствие ТЗ;
3) архитектуру и паттерны;
4) вклад участников;
5) качество инженерной дисциплины (тесты, обработка ошибок, документация).

## 2. Эволюция архитектуры (краткая хронология)

### Этап A: frontend-heavy реализация

Исторически ключевая логика сначала лежала в `src/frontend/*`:
- модель схемы;
- команды undo/redo;
- контроллер;
- builder для XML.

Плюс: быстрый старт разработки GUI.  
Минус: слабое разделение слоёв, backend фактически был каркасом.

### Этап B: backend-first реорганизация

Проведено выделение рабочей логики в backend:
- `src/backend/model/circuit.py`
- `src/backend/commands/history.py`
- `src/backend/application/app_controller.py`
- `src/backend/io/xml_builder.py`
- `src/backend/logic/truth_table.py`

GUI переведён на backend API.

### Этап C: прикладная стабилизация UX

После реорганизации внесены точечные улучшения:
- безмышечная навигация по доске;
- понятный режим соединения;
- явное удаление блоков;
- безопасное поведение таблицы истинности на неполной схеме (без падений приложения).

## 3. Детальная карта текущей реализации

## 3.1 Backend: модель и валидация схемы

Файл: `src/backend/model/circuit.py`

Реализовано:
- хранение узлов и связей;
- список допустимых типов узлов;
- получение числа входных/выходных пинов по типу;
- добавление/удаление узлов;
- соединение/разъединение пинов;
- проверка корректности соединения;
- проверка наличия циклов;
- структурная валидация схемы.

Что это даёт:
- схема контролируется на уровне модели, а не только UI;
- ошибки ловятся рано и в одном месте;
- backend устойчивее к невалидным состояниям.

## 3.2 Backend: команды и история

Файл: `src/backend/commands/history.py`

Реализовано:
- `CommandHistory` со стеками undo/redo;
- команды:
  - `AddNodeCommand`
  - `RemoveNodeCommand`
  - `MoveNodeCommand`
  - `ConnectPinsCommand`
  - `DisconnectPinsCommand`

Назначение:
- обеспечить воспроизводимые do/undo/redo операции;
- хранить не только факт действия, но и его контекст (для отката).

## 3.3 Backend: приложение/фасад

Файл: `src/backend/application/app_controller.py`

Реализовано:
- единая точка входа для GUI;
- вызовы создания/редактирования;
- интеграция с truth-table;
- интеграция с XML builder;
- обработка ошибок в ответах для UI.

Назначение:
- изолировать GUI от внутренней структуры backend;
- держать API стабильным при изменениях модели.

## 3.4 Backend: XML Builder

Файл: `src/backend/io/xml_builder.py`

Реализовано:
- экспорт схемы в XML;
- импорт схемы из XML;
- валидация связей при загрузке.

Плюс для защиты:
- легко показать, что формат файла не “декоративный”, а реально рабочий.

## 3.5 Backend: truth-table и вычисления

Файл: `src/backend/logic/truth_table.py`

Реализовано:
- вычисление таблицы истинности для схемы;
- вычисление таблицы для отдельного узла (на уровне логики);
- определение влияющих узлов;
- вычислительные функции для упрощения/полиномов (присутствуют в backend).

Особенности:
- перед вычислениями выполняется структурная валидация;
- ошибки в невалидной схеме теперь транслируются в UI-понятный формат.

## 4. Детальная карта Frontend (актуальное состояние)

## 4.1 Окно и действия

Файл: `src/frontend/app/main_window.py`

Реализовано:
- меню `Файл/Правка/Вид`;
- тулбар элементов и режимов;
- таблица истинности для всей схемы;
- диалог “Подсказка по управлению”;
- удаление блоков через action + статус-сообщения.

## 4.2 Сцена схемы

Файл: `src/frontend/canvas/scene.py`

Реализовано:
- графические объекты: узлы, пины, линии;
- режимы add/connect/disconnect;
- подсветка выбранного исходного пина при создании связи;
- удаление выбранных узлов (`delete_selected_nodes`).

## 4.3 Вьюпорт/навигация

Файл: `src/frontend/canvas/view.py`

Реализовано:
- zoom колесом;
- панорамирование средней кнопкой;
- клавиатурная навигация стрелками по доске;
- `+`/`-` для масштаба.
