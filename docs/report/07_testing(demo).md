# Тестирование

Тестирование проводилось на трёх уровнях: модульное, интеграционное и ручное. Целью тестирования является проверка корректности работы всех компонентов приложения, соответствия требованиям и выявление потенциальных ошибок.

## 1. Модульные тесты (Unit Tests)

Модульные тесты расположены в каталоге `tests/unit/`. Они проверяют каждый модуль изолированно, без взаимодействия с реальным GUI или файловой системой. Используется фреймворк `pytest`.

### Покрытие модулей

| Модуль | Тестируемые функции | Файл тестов |
|--------|---------------------|-------------|
| Модель схемы (`Circuit`) | Добавление/удаление узлов, добавление/удаление связей, поиск узла, позиционирование | `test_circuit.py` |
| Команды (`AddNodeCommand`, `RemoveNodeCommand`, `MoveNodeCommand`, `ConnectCommand`, `DisconnectCommand`) | Выполнение и отмена операций | `test_commands.py` |
| История команд (`CommandHistory`) | Стек undo/redo, очистка | `test_history.py` |
| Валидация | Проверка циклов, числа входов/выходов, существования узлов | `test_validation.py` |
| XML Builder | Сохранение схемы в XML, загрузка из XML, обработка ошибок формата | `test_io.py` |
| Логика | Таблица истинности, упрощение, полиномы | `test_logic.py` |

### Пример теста

```python
# tests/unit/test_circuit.py
import pytest
from backend.model.circuit import Circuit

def test_add_node():
    circuit = Circuit()
    node_id = circuit.add_node("AND", 100, 200)
    node = circuit.get_node(node_id)
    assert node["type"] == "AND"
    assert node["x"] == 100
    assert node["y"] == 200

def test_connect_nodes():
    circuit = Circuit()
    n1 = circuit.add_node("INPUT", 0, 0)
    n2 = circuit.add_node("OUTPUT", 0, 0)
    assert circuit.connect_nodes(n1, n2) is True
    assert (n1, n2) in circuit.get_connections()

def test_connect_cycle_detection():
    circuit = Circuit()
    n1 = circuit.add_node("AND", 0, 0)
    n2 = circuit.add_node("OR", 0, 0)
    circuit.connect_nodes(n1, n2)
    # Попытка создать цикл: n2 -> n1
    assert circuit.connect_nodes(n2, n1) is False  # Цикл не должен быть допущен
```

## 2. Интеграционные тесты

Интеграционные тесты проверяют взаимодействие между модулями. Они находятся в `tests/integration/`.

### Сценарии интеграционного тестирования

- **Команда + модель + история**: проверка, что последовательность выполнения и отмены команд приводит к корректному состоянию модели.
- **XML + модель**: загрузка схемы из XML и её последующее сохранение должно дать эквивалентный XML (с точностью до порядка атрибутов).
- **Контроллер + модель + команды**: проверка, что вызовы методов контроллера приводят к ожидаемым изменениям в модели и истории.

## 3. Ручное тестирование GUI

Ручное тестирование проводится по чек-листу, охватывающему все пользовательские сценарии. Чек-лист хранится в `tests/manual/checklist.md`.

### Чек-лист ручного тестирования

- [ ] Запуск приложения – главное окно открывается без ошибок.
- [ ] Добавление всех типов элементов (AND, OR, XOR, EQUAL, INPUT, OUTPUT) – элементы появляются на сцене.
- [ ] Удаление элемента (выбор + Delete) – элемент и его связи удаляются.
- [ ] Перемещение элемента – связи обновляются.
- [ ] Соединение элементов – связь создаётся; проверка валидации: нельзя соединить элемент сам с собой, нельзя превысить число входов, нельзя создать цикл.
- [ ] Отключение связи – связь удаляется.
- [ ] Undo/Redo – после каждой операции отмена и повтор восстанавливают состояние.
- [ ] Таблица истинности:
    - Режим «Схема» – отображает правильные значения для всех выходов.
    - Режим «По элементу» – при выборе узла отображается его таблица, влияющие узлы подсвечиваются.
- [ ] Упрощение схемы:
    - Фиксация входов, кнопка «Оценить» – выводится количество удаляемых элементов.
    - Кнопка «Упростить» – схема упрощается, Undo восстанавливает исходную.
- [ ] Полиномы – для выбранного узла отображается полином.
- [ ] Настройки отображения – изменение цветов, размеров, шрифтов применяется к сцене и сохраняется после перезапуска.
- [ ] Сохранение и загрузка:
    - Сохранение схемы в XML-файл.
    - Загрузка сохранённого файла – схема восстанавливается.
    - Загрузка некорректного XML – выводится сообщение об ошибке.
- [ ] Кроссплатформенность – проверка на Windows и Linux (при наличии).

## 4. Тестовые данные (фикстуры)

В каталоге `tests/fixtures/` подготовлены тестовые XML-файлы:

| Файл | Описание |
|------|----------|
| `simple_and.xml` | Корректная схема: два входа, AND, выход. |
| `xor_circuit.xml` | Схема с XOR (полусумматор). |
| `cycle.xml` | Схема с циклом (некорректная) для проверки валидации. |
| `missing_node.xml` | Связь ссылается на несуществующий узел. |
| `invalid_type.xml` | Элемент с неизвестным типом. |

## 5. Запуск тестов

Для запуска всех тестов используйте команду:

```bash
pytest tests/
```

Для запуска только модульных тестов:

```bash
pytest tests/unit/
```

Для запуска с подробным выводом:

```bash
pytest -v tests/
```
