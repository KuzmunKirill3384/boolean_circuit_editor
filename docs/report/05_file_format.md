# Формат файла схемы (XML)

## Общая структура

Корневой элемент: `<circuit>`

Внутри:
- `<nodes>` – список узлов схемы
- `<connections>` – список связей между узлами

## Секция `<nodes>`

Каждый узел описывается тегом `<node>` со следующими атрибутами:

| Атрибут | Тип | Описание |
|---------|------|----------|
| `id` | целое число | Уникальный идентификатор узла |
| `type` | строка | Тип элемента (AND, OR, XOR, EQUAL, INPUT, OUTPUT) |
| `x` | вещественное число | Координата x на рабочей области (пиксели) |
| `y` | вещественное число | Координата y на рабочей области (пиксели) |

Пример:
```xml
<nodes>
  <node id="1" type="INPUT" x="40.0" y="120.0"/>
  <node id="2" type="AND" x="200.0" y="170.0"/>
  <node id="3" type="OUTPUT" x="360.0" y="170.0"/>
</nodes>
```

## Секция `<connections>`

Каждая связь описывается тегом `<connection>` со следующими атрибутами:

| Атрибут | Тип | Описание |
|---------|------|----------|
| `from_node` | целое число | Идентификатор узла-источника (выход) |
| `to_node` | целое число | Идентификатор узла-приёмника (вход) |

Пример:
```xml
<connections>
  <connection from_node="1" to_node="2"/>
  <connection from_node="2" to_node="3"/>
</connections>
```

## Полный пример

```xml
<?xml version="1.0" encoding="UTF-8"?>
<circuit>
  <nodes>
    <node id="0" type="INPUT" x="40.0" y="120.0"/>
    <node id="1" type="INPUT" x="40.0" y="220.0"/>
    <node id="2" type="AND" x="200.0" y="170.0"/>
    <node id="3" type="OUTPUT" x="360.0" y="170.0"/>
  </nodes>
  <connections>
    <connection from_node="0" to_node="2"/>
    <connection from_node="1" to_node="2"/>
    <connection from_node="2" to_node="3"/>
  </connections>
</circuit>
```

## Примечания

- **Кодировка**: UTF-8.
- **Обязательность**: все атрибуты являются обязательными.
- **Валидация**: при загрузке файла необходимо проверять существование всех узлов, на которые ссылаются связи. Если связь ссылается на несуществующий узел, загрузка должна завершаться с ошибкой.
- **Расширяемость**: в будущем возможны дополнения, например, добавление атрибута `version` к корневому элементу или поддержка дополнительных атрибутов узла (метаданные, цвет и т.п.). Такие изменения будут обратно совместимы (старые файлы будут читаться, новые атрибуты игнорироваться) или потребуют новой версии формата.
