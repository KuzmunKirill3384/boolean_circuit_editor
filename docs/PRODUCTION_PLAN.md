# Производственный план: от и до

Стек: **Python 3** + **GUI на Python** (PyQt6 или Tkinter — см. этап 0). План от настройки окружения до сдачи.

Актуальная внутренняя структура репозитория и границы модулей описаны отдельно в `docs/ARCHITECTURE.md`.

---

## Стек и инструменты

| Назначение | Вариант | Примечание |
|------------|---------|------------|
| Язык | Python 3.10+ | — |
| GUI | **PyQt6** (рекомендуется) или Tkinter | PyQt: удобный QGraphicsScene для схемы, таблицы, диалоги |
| XML | `xml.etree.ElementTree` (stdlib) или `lxml` | Для Builder загрузки/сохранения |
| Тесты | `pytest` | Юнит и интеграционные |
| Окружение | `venv` + `requirements.txt` | — |

## Фаза 0: Старт (все, 1–2 дня)

### 0.1 Окружение и репозиторий

- [ ] Установить Python 3.10+.
- [ ] Клонировать репозиторий, создать виртуальное окружение:
  ```bash
  python -m venv .venv
  source .venv/bin/activate   # Windows: .venv\Scripts\activate
  ```
- [ ] Выбрать GUI: **PyQt6** или Tkinter. Добавить в `requirements.txt`:
  - PyQt6: `PyQt6>=6.4`
  - или использовать встроенный `tkinter`.
- [ ] Добавить в `requirements.txt`: `pytest`, при необходимости `lxml`.
- [ ] Установить: `pip install -r requirements.txt`.
- [ ] Создать структуру каталогов (см. ниже).

### 0.3 Контракты (все)

- [ ] **Интерфейс узла (Composite):** абстрактный класс/протокол с `id`, `accept(visitor)`, методы для входов/выходов (или общий интерфейс для обхода). Кузьмин предлагает, все согласовывают.
- [ ] **Перечень команд:** AddNode, RemoveNode, Connect, Disconnect, MoveNode. Сигнатуры — Кузьмин, Иван ориентируется для вызовов из UI.
- [ ] **Формат XML:** черновик в `docs/FILE_FORMAT.md` (Боос + Кузьмин): корень, список узлов (id, type, x, y), список связей (from_id, to_id, from_pin, to_pin).

### 0.4 Точка входа

- [ ] Создать `run.py`: импорт `frontend.main`, запуск приложения (пустое окно с заголовком). Убедиться, что `python run.py` открывает окно.

**Итог фазы 0:** репо клонирован, venv работает, структура папок создана, контракты зафиксированы, пустое окно запускается.

## Фаза 1: Бэкенд (Кузьмин)

Порядок: модель → валидация → команды/Visitor/Factory → Builder → логика. После каждого блока — коммит, при необходимости краткое описание API для Ивана и Бооса.

### 1.1 Модель (Composite)

- [ ] `backend/model/node.py`: абстрактный базовый класс узла (id, тип), метод `accept(visitor)`.
- [ ] `backend/model/elements.py`: классы Input, Output, And, Or, Xor, Equal (фиксированное число входов/выходов).
- [ ] `backend/model/circuit.py`: класс Circuit — контейнер узлов и списка связей (пары (node_id, pin) → (node_id, pin)); методы добавления/удаления связей, получение узла по id.
- [ ] Проверка: создать схему из 2 входов и AND, связать, обойти Visitor’ом-заглушкой.

### 1.2 Валидация

- [ ] `backend/validation/validator.py`: проверка циклов (обход в глубину от выходов к входам или наоборот), проверка числа входов/выходов при добавлении связи.
- [ ] API: `validate_before_connect(...)` / `validate_circuit(circuit)` — возврат списка ошибок (строками для UI).
- [ ] Покрыть тестами (Боос может подключиться): цикл, лишние связи.

### 1.3 Команды, Factory, Visitor

- [ ] `backend/commands/base.py`: абстрактная Command с `execute()`, `undo()`.
- [ ] `backend/visitor/visitor.py`: базовый Visitor, при необходимости конкретные (например, для копирования состояния в команде undo).
- [ ] `backend/commands/concrete.py`: AddNode, RemoveNode, Connect, Disconnect, MoveNode. Внутри — вызовы к Circuit и при необходимости Visitor.
- [ ] `backend/commands/factory.py`: CommandFactory.create_add_node(...), create_connect(...) и т.д.
- [ ] История: класс CommandHistory (стек undo, при необходимости redo), вызов execute при выполнении, undo при отмене.
- [ ] Проверка: выполнить несколько команд, откатить — состояние схемы восстанавливается.

### 1.4 Загрузка/сохранение (Builder)

- [ ] Спецификация XML в `docs/FILE_FORMAT.md` зафиксирована (Боос + Кузьмин).
- [ ] `backend/io/xml_builder.py`: построение Circuit из XML (Builder), сериализация Circuit в XML. Учитывать позиции узлов (x, y) для отображения.
- [ ] Тесты: сохранить простую схему, загрузить, сравнить набор узлов и связей.

### 1.5 Логика: таблица истинности, упрощение, полиномы

- [ ] `backend/logic/truth_table.py`: по Circuit построить таблицу истинности (входы + все выходы элементов или только выходы схемы); для выбранного узла — таблица по входам схемы и значению этого узла; список id узлов, от которых зависит выбранный (обход графа).
- [ ] `backend/logic/simplify.py`: при заданных значениях на входах — упростить схему (удалить неиспользуемые узлы, константы); оценка «сколько элементов можно убрать» для каждого входа или комбинации (перебор или эвристика — описать в отчёте).
- [ ] `backend/logic/polynomial.py`: полином Жегалкина (или иное полиномиальное представление) для каждого элемента — по таблице истинности выхода этого элемента.
- [ ] API, удобный для вызова из UI: функции или класс-фасад с методами get_truth_table(circuit), get_truth_table_for_node(circuit, node_id), get_affected_nodes(circuit, node_id), simplify(circuit, input_values), get_removable_count_per_input(...), get_polynomials(circuit).

**Итог фазы 1:** бэк готов: модель, валидация, команды с undo, загрузка/сохранение XML, расчёты. API стабилен для фронта и тестов.

## Фаза 2: Фронтенд (Иван)

Параллельно с фазой 1 после контрактов: сначала каркас и канвас, затем привязка к бэку по мере готовности API.

### 2.1 Каркас приложения

- [ ] Главное окно: меню (Файл — Открыть/Сохранить/Выход; Правка — Отмена; Вид — Таблица истинности, Упрощение, Полиномы, Настройки).
- [ ] Разделение на области: панель инструментов (кнопки добавления AND, OR, XOR, EQUAL), рабочая область схемы, по необходимости боковые панели (таблица истинности, упрощение, полиномы).
- [ ] Подключение к Circuit и CommandHistory (глобально или через контейнер/контроллер): при «Открыть»/«Сохранить» вызывать бэк (после готовности Builder).

### 2.2 Рабочая область схемы

- [ ] Виджет схемы: PyQt6 — `QGraphicsView`/`QGraphicsScene` с кастомными `QGraphicsItem` для узлов и линий; Tkinter — `Canvas` с примитивами и привязкой к координатам из модели.
- [ ] Отрисовка узлов по типам (AND, OR, XOR, EQUAL, входы/выходы) и линий связей по списку из Circuit. Координаты брать из модели (после сохранения в модель при Move).
- [ ] Добавление элемента: выбор типа → клик на сцене → создание команды AddNode с (type, x, y) → execute → перерисовка.
- [ ] Связь: выбор «режим связи» → клик на выход, затем на вход → валидация (Validator) → команда Connect → execute → перерисовка.
- [ ] Удаление: выбор элемента (и при необходимости связи) → команды RemoveNode/Disconnect → execute.
- [ ] Перемещение: перетаскивание узла → команда MoveNode с новыми (x, y) (или пакетное обновление позиции в конце drag).
- [ ] Кнопки Undo/Redo в меню или панели → вызов CommandHistory.undo()/redo() → обновление схемы.

### 2.3 Таблица истинности

- [ ] Панель/диалог «Таблица истинности»: запрос к backend logic — get_truth_table(circuit); отображение в таблице (QTableWidget / ttk.Treeview).
- [ ] Режим «по элементу»: выбор узла на схеме → get_truth_table_for_node(circuit, node_id), get_affected_nodes(...) → подсветка этих узлов на схеме другим цветом, таблица в панели.

### 2.4 Упрощение схемы

- [ ] Панель ввода: фиксированные значения на входах (список входов + 0/1 для каждого).
- [ ] Кнопка «Оценить» / «Сколько убрать»: вызов get_removable_count_per_input(...) или аналога → отображение результата.
- [ ] Кнопка «Упростить»: вызов simplify(circuit, input_values), замена текущей схемы на упрощённую (через команды или подмена Circuit), обновление отображения.

### 2.5 Полиномы

- [ ] При выборе узла на схеме (или в списке) показывать полином: get_polynomials(circuit) или get_polynomial_for_node(circuit, node_id) → отображение в панели/всплывающей подсказке/отдельном окне.

### 2.6 Настройки отображения

- [ ] Диалог/панель настроек: цвета элементов, линий, фона; размер узлов; шрифт подписей. Сохранение в конфиг (файл или QSettings) и применение к отрисовке.

### 2.7 Сохранение и загрузка

- [ ] «Сохранить в XML»: диалог выбора файла → вызов Builder (export Circuit to XML) → запись в файл. Обработка ошибок, сообщение пользователю.
- [ ] «Загрузить из XML»: диалог выбора файла → чтение → Builder (build Circuit from XML) → подмена текущей схемы и истории (или сброс истории), обновление отображения.

**Итог фазы 2:** полный UI по ТЗ, все действия идут через бэк, схема рисуется и редактируется, таблица истинности, упрощение и полиномы отображаются.
