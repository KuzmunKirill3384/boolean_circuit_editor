# Формат файла схемы (XML)

Спецификацию формата ведет **Боос Д. А.** в согласовании с **Кузьминым К. В.**. Этот документ фиксирует базовый XML-формат хранения схемы так, чтобы фронт, бэк и тесты работали с одним и тем же представлением.

## Назначение

XML-файл должен хранить:

- состав схемы;
- тип каждого элемента;
- идентификаторы узлов;
- связи между выходами и входами;
- координаты элементов на рабочей области;
- метаданные файла;
- при необходимости настройки отображения, относящиеся именно к схеме.

Формат должен быть:

- человекочитаемым;
- устойчивым к расширению;
- пригодным для валидации;
- достаточно простым для реализации через Builder.

## Общая структура

Корневой элемент:

```xml
<boolean-circuit version="1.0">
  <meta />
  <elements />
  <connections />
  <layout />
</boolean-circuit>
```

Обязательные секции:

- `meta` — служебная информация о файле;
- `elements` — все узлы схемы;
- `connections` — связи между узлами;
- `layout` — визуальные координаты и параметры размещения.

## Корневой элемент

Корневой тег: `boolean-circuit`

Обязательные атрибуты:

- `version` — версия формата, например `1.0`

Рекомендуемые атрибуты в будущем:

- `appVersion` — версия приложения;
- `createdAt` — дата создания;
- `updatedAt` — дата последнего сохранения.

## Секция `meta`

Секция хранит общие сведения о схеме.

Пример:

```xml
<meta>
  <title>Half Adder</title>
  <description>Пример полусумматора</description>
  <author>Team Boolean</author>
</meta>
```

Поля:

- `title` — человекочитаемое название схемы;
- `description` — краткое описание;
- `author` — автор или группа.

Все поля в `meta` являются необязательными.

## Секция `elements`

Каждый элемент описывается отдельным тегом `element`.

Пример:

```xml
<elements>
  <element id="in1" type="INPUT" label="A" inputs="0" outputs="1" />
  <element id="in2" type="INPUT" label="B" inputs="0" outputs="1" />
  <element id="xor1" type="XOR" label="XOR1" inputs="2" outputs="1" />
  <element id="out1" type="OUTPUT" label="S" inputs="1" outputs="0" />
</elements>
```

Обязательные атрибуты элемента:

- `id` — уникальный идентификатор в пределах схемы;
- `type` — тип элемента.

Рекомендуемые атрибуты:

- `label` — подпись для UI;
- `inputs` — число входов;
- `outputs` — число выходов.

Допустимые типы на текущем этапе:

- `INPUT`
- `OUTPUT`
- `AND`
- `OR`
- `XOR`
- `EQUAL`

Правила:

- `id` не должен повторяться;
- `type` должен принадлежать поддерживаемому списку;
- `inputs` и `outputs` должны соответствовать типу элемента;
- для `INPUT` количество входов равно `0`;
- для `OUTPUT` количество выходов равно `0`;
- для логических элементов количество выходов по умолчанию равно `1`.

## Секция `connections`

Каждая связь описывается тегом `connection`.

Пример:

```xml
<connections>
  <connection fromElement="in1" fromPin="0" toElement="xor1" toPin="0" />
  <connection fromElement="in2" fromPin="0" toElement="xor1" toPin="1" />
  <connection fromElement="xor1" fromPin="0" toElement="out1" toPin="0" />
</connections>
```

Обязательные атрибуты:

- `fromElement` — источник;
- `fromPin` — индекс выходного пина источника;
- `toElement` — приемник;
- `toPin` — индекс входного пина приемника.

Правила валидации:

- источник и приемник должны ссылаться на существующие элементы;
- индексы пинов должны находиться в допустимых диапазонах;
- один входной пин не должен получать более одной связи;
- связь не должна создавать цикл;
- прямое соединение элемента с самим собой запрещено, если оно ведет к циклу.

## Секция `layout`

Секция отвечает за расположение элементов на холсте.

Пример:

```xml
<layout>
  <position elementId="in1" x="40" y="120" />
  <position elementId="in2" x="40" y="220" />
  <position elementId="xor1" x="220" y="170" />
  <position elementId="out1" x="420" y="170" />
</layout>
```

Обязательные атрибуты:

- `elementId` — ссылка на элемент;
- `x` — координата по горизонтали;
- `y` — координата по вертикали.

Правила:

- каждая позиция должна ссылаться на существующий элемент;
- координаты хранятся в пикселях;
- если позиция отсутствует, UI может использовать координаты по умолчанию.

## Что не хранится в XML схемы

Чтобы не смешивать файл схемы и пользовательские предпочтения, в XML схемы не рекомендуется сохранять:

- глобальные настройки приложения;
- тему интерфейса для всех проектов;
- состояние последних открытых окон;
- историю Undo/Redo.

Это должно лежать отдельно в конфиге приложения.

## Полный пример

```xml
<?xml version="1.0" encoding="UTF-8"?>
<boolean-circuit version="1.0">
  <meta>
    <title>Half Adder</title>
    <description>Сумма двух булевых входов</description>
    <author>Boolean Circuit Editor Team</author>
  </meta>

  <elements>
    <element id="in1" type="INPUT" label="A" inputs="0" outputs="1" />
    <element id="in2" type="INPUT" label="B" inputs="0" outputs="1" />
    <element id="xor1" type="XOR" label="X1" inputs="2" outputs="1" />
    <element id="out1" type="OUTPUT" label="S" inputs="1" outputs="0" />
  </elements>

  <connections>
    <connection fromElement="in1" fromPin="0" toElement="xor1" toPin="0" />
    <connection fromElement="in2" fromPin="0" toElement="xor1" toPin="1" />
    <connection fromElement="xor1" fromPin="0" toElement="out1" toPin="0" />
  </connections>

  <layout>
    <position elementId="in1" x="40" y="120" />
    <position elementId="in2" x="40" y="220" />
    <position elementId="xor1" x="220" y="170" />
    <position elementId="out1" x="420" y="170" />
  </layout>
</boolean-circuit>
```

## Версионирование

Текущая версия формата: `1.0`

Правила:

- изменения, не ломающие совместимость, оформляются как минорные;
- изменения структуры, несовместимые со старыми файлами, требуют новой мажорной версии;
- Builder должен ориентироваться на `version` и при необходимости явно сообщать о неподдерживаемом формате.

## Ошибки формата, которые должен ловить загрузчик

- отсутствует корневой тег `boolean-circuit`;
- отсутствует атрибут `version`;
- нет секции `elements` или `connections`;
- повторяются `id` элементов;
- связь ссылается на несуществующий элемент;
- тип элемента не поддерживается;
- индексы пинов выходят за диапазон;
- схема содержит цикл;
- обязательные атрибуты отсутствуют.

## Примечания по реализации

- Кодировка файла: `UTF-8`.
- Для базовой версии достаточно `xml.etree.ElementTree`.
- При усложнении валидации допустимо перейти на `lxml`.
- Окончательный вариант формата должен быть отражен и в отчете, и в тестах загрузки/сохранения.
